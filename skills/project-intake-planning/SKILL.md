# Project Intake Planning

## Goal

Turn incoming requests into executable plans with clear boundaries and validation steps.

## Workflow

1. Read `docs/ai-kb/project-context.md` and `docs/ai-kb/architecture-map.md`.
2. Classify the request type: feature, bugfix, refactor, docs, or ops.
3. Define scope, non-goals, and impacted modules.
4. List assumptions, dependencies, and risks.
5. Build an ordered implementation plan with validation checkpoints.
6. Hand off to one execution skill and track follow-up items.

## Required Outputs

- One-paragraph task summary
- Impacted paths list
- Step-by-step implementation sequence
- Validation plan
- Open questions or assumptions

## References

- Use `references/intake-checklist.md` before finalizing the plan.
