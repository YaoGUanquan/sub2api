# Knowledge Base Maintenance

## Goal

Keep project memory concise, searchable, and useful for future coding sessions.

## Workflow

1. Read existing entries in `docs/ai-kb/` relevant to the current request.
2. Add new entries using existing templates and date format.
3. Capture rationale, not only outcomes.
4. Keep each entry concise and link affected paths when possible.
5. Move outdated execution docs to `docs/archive/` with an archive reason.

## Quality Rules

- Prefer factual summaries over narrative logs.
- Avoid duplicate entries across files.
- Use stable headings for quick grep-based discovery.
