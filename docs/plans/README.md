# Plans

Store active project plans and implementation proposals here.

## Recommended Naming

Use `YYYY-MM-DD-topic.md` for easy sorting.

## Reusable Templates

- `TEMPLATE.md`: generic implementation plan template
- `PR_WORKFLOW_TEMPLATE.md`: pull request workflow and quality checklist

## Lifecycle

1. Create a new plan from `TEMPLATE.md`.
2. Update status while executing.
3. Move completed or obsolete plans to `../archive/`.
